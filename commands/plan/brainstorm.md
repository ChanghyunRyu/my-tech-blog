# /brainstorm

아이디어를 탐색하고 옵션을 비교합니다.

## 사용법

```
/brainstorm 상태 관리 방법
/brainstorm 인증 구현 방식
/brainstorm 데이터베이스 선택
```

## 동작

1. 문제/목표 명확화
2. 2-3개 접근법 제시
3. 각 옵션의 장단점 분석
4. 권장안 명시
5. 사용자 확인 후 진행

## 출력 형식

```markdown
## 브레인스토밍: {주제}

### 옵션 A: {이름}
**장점**: ...
**단점**: ...
**적합한 경우**: ...

### 옵션 B: {이름}
**장점**: ...
**단점**: ...
**적합한 경우**: ...

### 옵션 C: {이름} (권장)
**장점**: ...
**단점**: ...
**적합한 경우**: ...

### 권장 이유
{왜 C를 권장하는지}

### 다음 단계
이 방향으로 진행할까요?
```

## 원칙

- 한 번에 하나의 질문만
- 200-300 단어 단위로 검증
- "이게 맞나요?" 확인 후 진행
