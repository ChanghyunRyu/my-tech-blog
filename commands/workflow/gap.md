# /gap

설계 문서와 구현 코드 간의 갭을 분석합니다.

## 사용법

```
/gap                       # 전체 분석
/gap docs/plans/auth/      # 특정 설계 문서
/gap --detail              # 상세 분석
```

## 동작

1. 설계 문서 파싱
2. 구현 코드 분석
3. 일치율 계산
4. 불일치 항목 보고

## 분석 항목

### 1. 컴포넌트 존재 여부
- 설계에 명시된 컴포넌트가 구현되었는가?
- 설계에 없는 컴포넌트가 추가되었는가?

### 2. 인터페이스 일치
- API 시그니처가 설계와 일치하는가?
- 반환 타입이 일치하는가?

### 3. 데이터 모델
- 스키마가 설계와 일치하는가?
- 필드 타입이 일치하는가?

### 4. 비기능 요구사항
- 성능 요구사항 충족?
- 보안 요구사항 충족?

## 출력 형식

```markdown
## 갭 분석 결과

### 요약
- **일치율**: 85%
- **상태**: needs_improvement
- **권장**: 1회 반복 개선

### 불일치 항목

#### Missing (설계에 있으나 구현 없음)
| 항목 | 설계 위치 | 심각도 |
|------|----------|--------|
| UserAuthService | auth.md:15 | High |

#### Mismatch (구현이 설계와 다름)
| 항목 | 설계 | 구현 | 심각도 |
|------|------|------|--------|
| createUser | (email, pwd) | (dto) | Medium |

#### Extra (설계에 없는 구현)
| 항목 | 구현 위치 | 조치 |
|------|----------|------|
| tempHelper | utils.py:45 | 문서화 또는 제거 |
```

## 옵션

| 플래그 | 효과 |
|--------|------|
| `--detail` | 상세 분석 |
| `--json` | JSON 형식 출력 |

## 연관 커맨드
