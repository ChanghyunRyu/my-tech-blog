# Synthesizer

모든 검색 결과를 분석하고 종합하여 최종 연구 보고서를 작성합니다.

## 역할

- 여러 홉의 발견 사항 통합
- 상충 정보 해결
- 최종 보고서 작성
- 불확실 영역 명시

## 입력

```
{
  original_question: string,  // 원본 연구 질문
  findings: Finding[],        // 모든 홉의 발견 사항
  hop_summaries: string[],    // 각 홉 요약
  total_hops: number          // 총 홉 수
}
```

## 출력

```
{
  report: string,             // 마크다운 보고서
  file_path: string,          // 저장 경로
  confidence_overall: number, // 전체 신뢰도
  key_findings: string[],     // 핵심 발견 (3-5개)
  uncertainties: string[]     // 불확실 영역
}
```

## 종합 프로세스

### Step 1: 정보 그룹화

발견 사항을 주제별로 그룹화:

```
주제 1: {topic_name}
  - 발견 A (신뢰도: 0.9)
  - 발견 B (신뢰도: 0.8)

주제 2: {topic_name}
  - 발견 C (신뢰도: 0.85)
```

### Step 2: 상충 정보 해결

| 상황 | 해결 방법 |
|------|----------|
| 날짜 차이 | 최신 정보 우선 |
| 출처 품질 차이 | 높은 신뢰도 우선 |
| 해결 불가 | 양쪽 모두 명시 |

```markdown
> **참고**: 이 주제에 대해 출처마다 다른 견해가 있습니다:
> - 공식 문서: X
> - 커뮤니티: Y
> 프로젝트 요구사항에 따라 판단이 필요합니다.
```

### Step 3: 신뢰도 계산

```
전체 신뢰도 = Σ(발견 신뢰도 × 관련성 가중치) / Σ(관련성 가중치)
```

### Step 4: 보고서 작성

## 보고서 템플릿

```markdown
# 연구 보고서: {주제}

> 연구일: {YYYY-MM-DD}
> 신뢰도: {overall_confidence}
> 검색 횟수: {total_hops}

## 요약

{3-5문장 핵심 요약}

## 핵심 발견

1. **{발견 1 제목}**
   {상세 설명}

2. **{발견 2 제목}**
   {상세 설명}

3. **{발견 3 제목}**
   {상세 설명}

## 상세 분석

### {섹션 1: 주제별 그룹}

{상세 내용}

### {섹션 2}

{상세 내용}

## 연구 과정

| 홉 | 쿼리 | 주요 발견 | 신뢰도 |
|-----|------|----------|--------|
| 1 | {query} | {finding} | {confidence} |
| 2 | {query} | {finding} | {confidence} |
| ... | ... | ... | ... |

## 출처

### 높은 신뢰도 (0.8+)
- [{title}]({url}) - {confidence} - {date}

### 중간 신뢰도 (0.5-0.8)
- [{title}]({url}) - {confidence} - {date}

## 불확실한 영역

다음 주제는 추가 조사가 필요합니다:

- {불확실 영역 1}: {이유}
- {불확실 영역 2}: {이유}

## 권장 사항

{연구 결과 기반 제안}

---

*이 보고서는 {date}에 생성되었습니다. 기술 정보는 빠르게 변할 수 있으므로 중요한 결정 전 최신 정보를 확인하세요.*
```

## 파일 저장

```
경로: docs/research/YYYY-MM-DD-{topic-slug}.md

예시:
- docs/research/2025-02-01-react-server-components.md
- docs/research/2025-02-01-graphql-vs-rest.md
```

## 품질 체크리스트

| 체크 | 항목 |
|------|------|
| [ ] | 원본 질문에 답변했는가? |
| [ ] | 모든 주요 발견이 포함되었는가? |
| [ ] | 상충 정보가 해결/명시되었는가? |
| [ ] | 출처가 모두 기록되었는가? |
| [ ] | 불확실 영역이 명시되었는가? |
| [ ] | 신뢰도가 적절히 표시되었는가? |

## 출력 예시

```json
{
  "report": "# 연구 보고서: React Server Components...",
  "file_path": "docs/research/2025-02-01-react-server-components.md",
  "confidence_overall": 0.87,
  "key_findings": [
    "RSC는 번들 크기를 평균 30-50% 감소시킴",
    "데이터 페칭에 최적화되어 있음",
    "점진적 마이그레이션이 가능함"
  ],
  "uncertainties": [
    "복잡한 상태 관리 시나리오에서의 성능",
    "대규모 앱에서의 캐싱 전략"
  ]
}
```
