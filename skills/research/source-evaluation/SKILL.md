# Source Evaluation Skill

## 목적
연구 결과의 출처를 평가하고 신뢰도 점수를 부여합니다.

## 신뢰도 점수 체계 (0.0 - 1.0)

| 점수 범위 | 의미 | 조치 |
|-----------|------|------|
| 0.9 - 1.0 | 매우 높음 | 결론에 포함, 핵심 근거로 사용 |
| 0.7 - 0.9 | 높음 | 결론에 포함 |
| 0.5 - 0.7 | 중간 | 추가 검증 권장 명시 |
| 0.3 - 0.5 | 낮음 | 불확실성 명시적 표기 |
| 0.0 - 0.3 | 매우 낮음 | 참고만, 결론에서 제외 |

## 출처 유형별 기본 신뢰도

### Tier 1: 공식 출처 (0.9+)
| 출처 | 신뢰도 | 예시 |
|------|--------|------|
| 공식 문서 | 0.95 | docs.python.org, react.dev |
| RFC/표준 문서 | 0.95 | IETF RFC, W3C 스펙 |
| 학술 논문 (피어 리뷰) | 0.90 | arXiv, ACM, IEEE |
| 공식 블로그 | 0.90 | engineering.fb.com |

### Tier 2: 신뢰할 수 있는 출처 (0.7-0.9)
| 출처 | 신뢰도 | 예시 |
|------|--------|------|
| 유명 기술 블로그 | 0.80 | kentcdodds.com, Dan Abramov |
| 핵심 기여자 글 | 0.85 | 라이브러리 메인테이너 |
| GitHub 공식 저장소 | 0.85 | README, 공식 예제 |
| Stack Overflow (검증됨) | 0.75 | 높은 투표, 수락된 답변 |

### Tier 3: 일반 출처 (0.5-0.7)
| 출처 | 신뢰도 | 예시 |
|------|--------|------|
| 일반 기술 블로그 | 0.60 | Medium, dev.to |
| 튜토리얼 사이트 | 0.55 | 다수 튜토리얼 사이트 |
| Stack Overflow (미검증) | 0.50 | 낮은 투표, 미수락 |
| 포럼 토론 | 0.50 | Reddit, Discord |

### Tier 4: 주의 필요 (0.3-0.5)
| 출처 | 신뢰도 | 예시 |
|------|--------|------|
| 오래된 문서 (2년+) | 0.40 | 기술 변화 가능성 |
| 익명 출처 | 0.35 | 저자 불명 |
| 자동 생성 콘텐츠 | 0.30 | AI 생성 의심 콘텐츠 |

## 신뢰도 조정 요인

### 신뢰도 상승 (+)
| 요인 | 조정값 | 설명 |
|------|--------|------|
| 최신 정보 (6개월 이내) | +0.10 | 기술 분야는 빠르게 변함 |
| 다수 출처 일치 | +0.10 | 교차 검증됨 |
| 코드 예제 포함 | +0.05 | 실용적 검증 가능 |
| 저자 명성/전문성 | +0.10 | 해당 분야 전문가 |
| 공식 출처 인용 | +0.05 | 1차 출처 참조 |

### 신뢰도 하락 (-)
| 요인 | 조정값 | 설명 |
|------|--------|------|
| 오래된 정보 (2년+) | -0.20 | 기술 변화 가능성 |
| 다른 출처와 상충 | -0.15 | 검증 필요 |
| 광고/스폰서 콘텐츠 | -0.10 | 편향 가능성 |
| 문법/오류 다수 | -0.10 | 품질 문제 |
| 출처 없는 주장 | -0.15 | 검증 불가 |

## 평가 체크리스트

| 항목 | 확인 내용 |
|------|----------|
| 저자 | 누가 작성했는가? 전문성은? |
| 날짜 | 언제 작성/업데이트되었는가? |
| 출처 | 주장의 근거가 있는가? |
| 목적 | 정보 제공? 판매? 의견? |
| 일관성 | 다른 출처와 일치하는가? |

## 상충 정보 처리

```markdown
### 상충 발견 시 처리

1. **양쪽 출처 신뢰도 비교**
   - 출처 A (공식 문서): 0.95
   - 출처 B (블로그): 0.70
   → 출처 A 우선

2. **시간순 고려**
   - 최신 정보가 이전 정보를 대체하는 경우 많음
   - 단, 공식 문서 업데이트 지연 고려

3. **명시적 불확실성 표기**
   "출처에 따라 다른 견해가 있습니다:
   - 공식 문서: X
   - 커뮤니티 의견: Y
   추가 검증이 권장됩니다."
```

```markdown
## 출처

### 불확실 (확인 필요)
- [Reddit 토론](https://reddit.com/...) - 0.45
```
