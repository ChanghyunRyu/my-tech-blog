# Introspection Skill

## 목적
메타인지 모드 - 추론 과정을 명시적으로 설명하고, 대안을 제시하며, 확신도를 표기합니다.

## 활성화
`--introspect` 플래그 사용 시 활성화

## 핵심 원칙

> "내가 왜 이 결정을 내렸는지 명확히 설명할 수 없다면, 그 결정은 충분히 검토되지 않은 것이다."

## 모든 결정에 포함할 요소

### 1. 결정 (Decision)
현재 선택한 옵션을 명확히 기술

### 2. 근거 (Reasoning)
그 결정을 내린 이유들을 나열

### 3. 대안들 (Alternatives)
고려한 다른 옵션들과 선택하지 않은 이유

### 4. 확신도 (Confidence)
결정에 대한 확신 수준과 그 이유

### 5. 가정 (Assumptions)
결정의 기반이 되는 전제들

## 출력 형식

```markdown
## [결정] React Query 사용

### 근거
- 서버 상태 관리에 특화됨
- 캐싱, 재시도, 배경 동기화 기본 제공
- 현재 프로젝트가 API 중심 구조

### 대안들
A) **Redux Toolkit Query**
   - 선택 안 함: 이미 Redux 미사용, 추가 보일러플레이트

B) **SWR**
   - 선택 안 함: React Query가 더 풍부한 기능 (mutation 등)

C) **직접 구현**
   - 선택 안 함: 시간 비용, 엣지 케이스 처리 부담

### 확신도: 높음 (0.85)
- 이유: 공식 문서에서 권장하는 사용 사례와 일치
- 팀 내 사용 경험 존재

### 가정
- API 응답 형식이 일관적임
- 오프라인 지원은 필요하지 않음
- 번들 크기 제약이 심하지 않음
```

## 확신도 기준

| 수준 | 점수 | 기준 | 표현 |
|------|------|------|------|
| 높음 | 0.8+ | 명확한 문서/사례 존재, 경험적 검증 가능 | "높은 확신" |
| 중간 | 0.5-0.8 | 합리적 추론이나 직접 검증 부재 | "중간 확신, 검증 권장" |
| 낮음 | 0.3-0.5 | 추측, 불확실성 높음 | "낮은 확신, 추가 조사 필요" |
| 매우 낮음 | 0.3 미만 | 정보 부족, 완전한 추측 | "확인 필수" |

## 확신도 영향 요인

### 확신도 상승
- 공식 문서 확인
- 실제 테스트/검증 완료
- 다수 출처 일치
- 과거 유사 경험
- 명확한 요구사항

### 확신도 하락
- 정보 출처 부족
- 상충되는 의견
- 새로운/익숙하지 않은 기술
- 모호한 요구사항
- 시간 제약으로 충분한 조사 불가

## 메타인지 질문

결정 전 자문할 질문들:

1. **왜 이것인가?**
   - 이 선택의 구체적인 이유는?

2. **왜 다른 것이 아닌가?**
   - 대안들을 충분히 고려했는가?

3. **내가 틀릴 수 있는 경우는?**
   - 이 결정이 잘못될 수 있는 시나리오는?

4. **무엇을 가정하고 있는가?**
   - 명시되지 않은 전제는?

5. **더 알아야 할 것은?**
   - 결정에 필요한 누락된 정보는?

## 불확실성 표현

```markdown
## 불확실한 영역

### 확인 필요
- [ ] Redis 클러스터 설정이 현재 인프라와 호환되는지
- [ ] 예상 트래픽에서의 성능 (벤치마크 필요)

### 가정한 사항
- 현재 인프라가 Redis 6.0+ 지원 (확인 필요)
- 메모리 2GB 이상 가용 (확인 필요)

### 리스크
- 캐시 무효화 전략이 복잡해질 수 있음
- 운영 복잡도 증가
```

## DO / DON'T

### DO
- 모든 주요 결정에 근거 명시
- 고려한 대안 나열
- 불확실한 부분 솔직히 인정
- 가정을 명시적으로 표현

### DON'T
- 결론만 제시하고 이유 생략
- "당연히", "명백히" 같은 표현
- 대안 없이 하나의 옵션만 제시
- 확신 없이 단정적으로 말하기
