# Tutorial 문서 작성 가이드

> 참고: [Diátaxis - Tutorials](https://diataxis.fr/tutorials/)

## 정의

Tutorial은 **학습자를 손잡고 이끄는 수업**입니다.
- 목적: 학습자가 무언가를 **할 수 있게** 만드는 것
- 비유: 운전 교습 — A에서 B로 가는 게 아니라, 운전 기술과 자신감을 키우는 것

## 핵심 원칙

### DO
- **학습자가 직접 해보게** 하라 (hands-on)
- **작동하는 결과**를 보여줘라 — 학습자는 성공 경험이 필요
- **최소한의 설명**만 — 이해보다 경험이 먼저
- **반복 가능**하게 — 같은 단계를 따르면 같은 결과
- **안전한 환경**에서 — 실패해도 괜찮은 샌드박스

### DON'T
- 선택지를 주지 마라 — "이렇게 하세요"가 맞음
- 대안을 설명하지 마라 — 혼란만 가중
- 추상적 개념을 먼저 설명하지 마라 — 경험 후 이해

## 구조 템플릿

```markdown
# [Tutorial 제목]: [달성할 것]

## 시작하기 전에
- 필요한 것: [사전 요구사항]
- 소요 시간: [예상 시간]
- 결과물: [완료 후 얻는 것]

## Step 1: [첫 번째 행동]
[구체적 지시]

## Step 2: [두 번째 행동]
[구체적 지시]

...

## 완료!
축하합니다. 이제 [달성한 것]을 할 수 있습니다.

## 다음 단계
- [How-to 가이드 링크]
- [더 깊은 Tutorial 링크]
```

## 품질 체크리스트

- [ ] 학습자가 **처음부터 끝까지 따라할 수 있는가**?
- [ ] 각 단계가 **하나의 행동**만 요구하는가?
- [ ] **작동하는 결과**를 확인할 수 있는가?
- [ ] 불필요한 설명이나 대안이 **없는가**?
- [ ] 실패 시 **복구 방법**이 있는가?

## 예시: 좋은 Tutorial vs 나쁜 Tutorial

### 나쁜 예
```
데이터베이스 연결에는 여러 방법이 있습니다.
PostgreSQL, MySQL, SQLite 중 선택할 수 있으며...
(선택지와 설명이 너무 많음)
```

### 좋은 예
```
## Step 3: 데이터베이스 연결

터미널에서 다음을 실행하세요:
$ python -c "from src.config import load_config; print('OK')"

'OK'가 출력되면 성공입니다.
```
