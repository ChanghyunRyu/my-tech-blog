# 태스크 관리 모드 (--task-manage)

작업을 계층적으로 분해하고 진행 상황을 체계적으로 추적하는 모드입니다.

## 활성화

```
/implement --task-manage 대시보드
"이 기능 구현해줘 --task-manage"
```

## 동작 원칙

### 1. 계층적 작업 분해

```markdown
## 작업 분해 구조

📦 Epic: 대시보드 구현
├── 🎯 Feature: 사용자 통계
│   ├── 📋 Task: API 엔드포인트 구현
│   │   ├── ✓ Subtask: 스키마 정의
│   │   ├── → Subtask: 컨트롤러 작성 (진행중)
│   │   └── ○ Subtask: 테스트 작성
│   └── 📋 Task: 차트 컴포넌트
│       └── ...
└── 🎯 Feature: 실시간 알림
    └── ...
```

### 2. 상태 시각화

```markdown
## 상태 표시

| 기호 | 상태 | 설명 |
|------|------|------|
| ○ | pending | 대기 중 |
| → | in_progress | 진행 중 |
| ✓ | completed | 완료 |
| ✗ | blocked | 차단됨 |
| ⏸ | paused | 일시 중지 |
```

### 3. 진행률 표시

```markdown
## 진행률 형식

### Feature: 사용자 통계
████████░░░░░░░░ 52% (5/9 tasks)

**완료**: 5 | **진행중**: 2 | **대기**: 2 | **차단**: 0
```

### 4. 의존성 관리

```markdown
## 의존성 그래프

Task A ──────────────────────→ Task D
           ↘                 ↗
Task B ──────→ Task C ──────┘
           ↗
Task E ────┘

**Critical Path**: A → D (5일)
**병렬 가능**: B, E
```

## 출력 형식

### 작업 계획

```markdown
# 📊 작업 계획: [프로젝트명]

## 개요
- **총 작업**: 15개
- **예상 기간**: 5일
- **우선순위**: 높음

## 작업 분해

### Epic 1: [이름]
상태: ░░░░░░░░░░ 0%

#### Feature 1.1: [이름]
| ID | Task | 상태 | 의존성 | 예상 |
|----|------|------|--------|------|
| T1 | [작업명] | ○ | - | 2h |
| T2 | [작업명] | ○ | T1 | 1h |
| T3 | [작업명] | ○ | T1, T2 | 3h |

### 의존성 맵
```
T1 → T2 ─┐
    ↘    ├→ T3
T4 ─────┘
```

## 시작 작업
먼저 시작할 수 있는 작업:
1. **T1**: [작업명] - 의존성 없음
2. **T4**: [작업명] - 의존성 없음
```

### 진행 상황 보고

```markdown
# 📈 진행 상황 보고

## 전체 현황
████████████░░░░ 75% 완료

| 상태 | 개수 | 비율 |
|------|------|------|
| ✓ 완료 | 12 | 75% |
| → 진행중 | 2 | 12.5% |
| ○ 대기 | 1 | 6.25% |
| ✗ 차단 | 1 | 6.25% |

## 최근 완료
- ✓ T8: API 엔드포인트 구현 (1시간 전)
- ✓ T7: 데이터 모델 정의 (2시간 전)

## 현재 진행 중
- → T9: 프론트엔드 컴포넌트 (50%)
- → T10: 테스트 작성 (30%)

## 차단된 작업
- ✗ T11: 배포 설정
  - **원인**: CI/CD 파이프라인 권한 필요
  - **해결 필요**: 관리자 승인

## 다음 작업
T9, T10 완료 후:
1. T11 차단 해제 대기
2. T12: 문서화 시작 가능
```

### 차단 해결

```markdown
# 🚧 차단 해결

## 차단된 작업: T11 - 배포 설정

### 원인 분석
- **직접 원인**: CI/CD 권한 없음
- **근본 원인**: 관리자 승인 프로세스 미완료

### 영향 범위
```
T11 (차단) → T12 (대기) → T13 (대기)
                          ↘ T14 (대기)
```
**영향받는 작업**: 4개

### 해결 옵션

1. **관리자 승인 요청** ⭐ 권장
   - 예상 시간: 1일
   - 리스크: 낮음

2. **로컬 테스트로 우회**
   - 예상 시간: 즉시
   - 리스크: 배포 검증 누락

### 권장 조치
관리자 승인을 요청하면서, 로컬 테스트로 T12-T14를 선진행
```

## 스킬 연동

`.claude/skills/practices/planning/` 스킬 활용:

```markdown
## 작업 분해 원칙
- SMART 기준 적용
- 2-8시간 단위로 분해
- 명확한 완료 조건
- 검증 가능한 산출물
```

## TodoWrite 연동

이 모드는 `TodoWrite` 도구와 자동 연동:

```markdown
## 자동 동기화
- 작업 상태 변경 → TodoWrite 업데이트
- 완료/차단 시 자동 반영
- 진행률 실시간 계산
```

## 주의사항

- 너무 세밀하게 분해하지 않기 (최소 30분 단위)
- 의존성은 실제로 필요한 것만 설정
- 차단 발생 시 즉시 보고
- 정기적으로 진행 상황 요약
