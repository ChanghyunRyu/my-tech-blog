# 브레인스토밍 모드 (--brainstorm)

아이디어를 탐색하고 다양한 옵션을 제시하는 창의적 모드입니다.

## 활성화

```
/implement --brainstorm 결제 시스템
"결제 시스템 설계해줘 --brainstorm"
```

## 동작 원칙

### 1. 한 번에 하나의 질문

```markdown
## 잘못된 예
"결제 시스템에 대해 여러 가지 질문이 있습니다.
1. 어떤 PG사를 사용할까요?
2. 결제 방식은요?
3. 환불 정책은요?"

## 올바른 예
"먼저 PG사 선택에 대해 이야기해볼까요?
현재 고려할 수 있는 옵션은..."
```

### 2. 2-3개 접근법 제시

```markdown
## 접근법 제시 형식

### 옵션 A: [이름] ⭐ 권장
**개요**: 간단한 설명
**장점**:
- 장점 1
- 장점 2
**단점**:
- 단점 1
**적합한 경우**: 언제 이 옵션이 좋은지

### 옵션 B: [이름]
...

### 옵션 C: [이름]
...

**권장 이유**: 옵션 A를 권장하는 구체적인 이유
```

### 3. 200-300 단어 단위 검증

```markdown
## 체크포인트

[200-300 단어 분량의 설명 후]

---
📋 **확인**: 지금까지의 내용이 맞나요?
- 요구사항: [요약]
- 선택한 방향: [선택]
- 다음 단계: [예정]

[계속] [수정 필요] [다른 방향]
```

### 4. 확인 후 진행

```markdown
## 진행 방식

1. 아이디어 제시 → "이게 맞나요?" 확인
2. 사용자 피드백 수집
3. 다음 아이디어로 이동 또는 수정
4. 최종 합의 도달
```

## 출력 형식

### 아이디어 탐색

```markdown
# 💡 [주제] 브레인스토밍

## 현재 이해
- 목표: [목표 요약]
- 제약: [제약 사항]
- 맥락: [관련 정보]

## 접근법 옵션

### 1. [접근법 A] ⭐ 권장
...

### 2. [접근법 B]
...

### 3. [접근법 C]
...

## 비교 요약

| 기준 | 옵션 A | 옵션 B | 옵션 C |
|------|--------|--------|--------|
| 복잡도 | 낮음 | 중간 | 높음 |
| 확장성 | 높음 | 중간 | 높음 |
| 구현 시간 | 짧음 | 중간 | 김 |

## 다음 질문

[다음으로 결정해야 할 사항]에 대해 어떻게 생각하시나요?
```

### 결정 요약

```markdown
# 📋 결정 요약

## 합의된 사항
1. [결정 1]
2. [결정 2]
3. [결정 3]

## 보류 사항
- [추후 결정할 사항]

## 다음 단계
→ [구체적인 다음 작업]

이 방향으로 진행할까요?
```

## 스킬 연동

`.claude/skills/practices/brainstorming/` 스킬과 연동:

```markdown
## 관련 스킬
- 아이디어 발산 기법
- 비교 분석 프레임워크
- 의사결정 매트릭스
```

## 주의사항

- 너무 많은 옵션을 한 번에 제시하지 않기 (최대 3개)
- 권장안은 반드시 명시
- 사용자 확인 없이 진행하지 않기
- 결정 사항은 문서화
