# 메타인지 모드 (--introspect)

추론 과정을 명시적으로 드러내고, 확신도와 가정을 표시하는 모드입니다.

## 활성화

```
/design --introspect API 설계
"이 버그 분석해줘 --introspect"
```

## 동작 원칙

### 1. 매 결정마다 근거 설명

```markdown
## 결정 형식

### 결정: [결정 내용]

**근거**:
1. [이유 1] - 출처/경험
2. [이유 2] - 출처/경험
3. [이유 3] - 출처/경험

**대안 고려**:
- [대안 A]: 선택하지 않은 이유
- [대안 B]: 선택하지 않은 이유
```

### 2. 확신도 표시

```markdown
## 확신도 레벨

### 🟢 높음 (High Confidence)
- 명확한 문서/코드 기반
- 검증된 패턴
- 직접 확인한 사실

### 🟡 중간 (Medium Confidence)
- 일반적인 베스트 프랙티스
- 유사 사례 기반 추론
- 부분적 정보

### 🔴 낮음 (Low Confidence)
- 추측 또는 가정
- 불완전한 정보
- 추가 검증 필요
```

### 3. 가정 명시

```markdown
## 가정 형식

⚠️ **가정**:
1. [가정 1] - 검증 방법: [방법]
2. [가정 2] - 검증 방법: [방법]

이 가정이 틀리다면:
- [영향 1]
- [영향 2]
```

### 4. 추론 과정 시각화

```markdown
## 추론 체인

[관찰] 코드에서 NullPointerException 발생
    ↓
[분석] user.getName() 호출 지점
    ↓ 🟡 중간 확신
[추론] user 객체가 null일 가능성
    ↓
[가정] DB 조회 실패 시 null 반환
    ↓ 🔴 검증 필요
[결론] null 체크 추가 또는 Optional 사용 권장
```

## 출력 형식

### 분석 보고

```markdown
# 🔍 [주제] 분석

## 현재 이해 🟢
[명확하게 파악된 사항]

## 추론 과정

### Step 1: [단계]
- 관찰: [사실]
- 해석: [해석] 🟡
- 근거: [근거]

### Step 2: [단계]
...

## 결론
[결론 내용]

### 확신도: 🟡 중간

**근거**:
1. [근거 1]
2. [근거 2]

**가정**:
⚠️ [가정 내용]

**검증 필요**:
- [ ] [검증 항목 1]
- [ ] [검증 항목 2]

## 대안적 해석

| 해석 | 확신도 | 조건 |
|------|--------|------|
| [해석 A] | 🟢 | [조건] |
| [해석 B] | 🟡 | [조건] |
| [해석 C] | 🔴 | [조건] |
```

### 결정 문서

```markdown
# 📝 결정 기록

## 결정 사항
[결정 내용]

## 결정 근거

### 분석된 옵션
1. **[옵션 A]** ✓ 선택
   - 장점: [...]
   - 단점: [...]
   - 확신도: 🟢

2. **[옵션 B]**
   - 선택하지 않은 이유: [...]

## 추론 과정

```
[전제 1] + [전제 2] → [중간 결론]
[중간 결론] + [전제 3] → [최종 결론]
```

## 가정과 리스크

### 가정
⚠️ [가정 1]: [내용]
⚠️ [가정 2]: [내용]

### 리스크
- 가정이 틀릴 경우: [영향]
- 완화 방안: [대안]

## 재검토 트리거
다음 상황에서 이 결정을 재검토:
- [트리거 1]
- [트리거 2]
```

## 스킬 연동

`.claude/skills/practices/introspection/` 스킬 활용:

```markdown
## 메타인지 체크리스트
- [ ] 모든 주요 결정에 근거 명시
- [ ] 확신도 레벨 표시
- [ ] 가정 식별 및 문서화
- [ ] 대안 고려 기록
- [ ] 검증 방법 제시
```

## 주의사항

- 단순한 사실은 확신도 표시 불필요
- 모든 문장에 표시하지 않기 (주요 결정/추론에만)
- 낮은 확신도일 때는 반드시 검증 방법 제시
- 가정이 많으면 먼저 검증하고 진행
